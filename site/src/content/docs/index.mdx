An eval suite is a YAML file plus a JSONL dataset. This one evaluates a `cat`
echo target, so it runs with no network and no API keys:

```sh
# Install
cargo install evalcore

# Run the shipped example (no API keys, no network)
evalcore run examples/quickstart/evals.yaml
```

```yaml
# evals.yaml
targets:
  echo:
    type: shell
    cmd: "cat"
datasets:
  - file: cases.jsonl
scorers:
  - type: contains
    value: "refund"
    case_sensitive: false
```

`evalcore run` exits `0` when every case passes and `1` otherwise — it drops
straight into CI.

		interface — you never write Rust.
	</Card>
	<Card title="Agent trajectories" icon="random">
		Ingest recorded OTel/OpenInference traces and grade **the answer and the
		path** in one suite: `must_call` / `must_not_call` ordering rules, step
		budgets, and final-answer grading.
	</Card>
	<Card title="Gates & baselines" icon="rocket">
		Block regressions, not imperfection. `--baseline` tolerates accepted

		mean-score floors over the whole run.
	</Card>

**No telemetry. No signup. Local SQLite.** Results live in `.evalcore/` next to
your repo — there is no server and nothing to sign up for.
