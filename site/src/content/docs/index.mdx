## How it works

Three steps: describe the suite as data, run it once against the real model, then
replay the recorded cassette in CI forever.

### <span class="step-n">1</span> Describe the suite in YAML

An eval is a YAML file plus a JSONL dataset — no SDK, no test harness to write.
Point a target at your app, list the datasets, and stack scorers. Here a `contains`
check and an [LLM-as-judge](/evalcore/guides/llm-as-judge/) grade every case:

```yaml
# evals.yaml
targets:
  support-bot:
    type: openai-compatible
    url: https://api.openai.com/v1
    model: gpt-4.1-mini
    api_key_env: OPENAI_API_KEY
datasets:
  - file: cases.jsonl
scorers:
  - type: contains
    value: refund
    case_sensitive: false
  - type: judge
    url: https://api.openai.com/v1
    model: gpt-4.1
    rubric: Is the answer grounded in the retrieved context?
    api_key_env: OPENAI_API_KEY
    threshold: 0.7
```

### <span class="step-n">2</span> Run it once, live

The first run has no cache hits, so every call goes live against the model and
gets recorded to a local SQLite cassette (the default `auto` mode: replay hits,
record misses). Token usage and cost appear on the summary when the target
declares `cost` rates:

```ansi
$ evalcore run evals.yaml
PASS refund-1 (843ms)
PASS refund-2 (512ms)

2 passed, 0 failed, 2 total · 2202 tokens · $0.0038
```

### <span class="step-n">3</span> Replay forever in CI — $0, offline, deterministic

Commit the cassette. Now CI runs `--cache replay`: every call is served from the
recording, so there is no network, no API key, and no flaky judge — calls return
instantly, and the recorded token and dollar totals are reported as virtual usage:

```ansi
$ evalcore run evals.yaml --cache replay --baseline main
PASS refund-1 (0ms)
PASS refund-2 (0ms)

2 passed, 0 failed, 2 total · 2202 tokens · $0.0038
```

`--baseline main` fails only on regressions, not on accepted failures. `evalcore run`
exits `0` when everything passes and `1` otherwise, so CI gates on it directly:

```yaml
# .github/workflows/evals.yml
name: evals
on: pull_request
jobs:
  evals:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: eval-core/evalcore@v0.5.0
        with:
          config: evals.yaml
          args: --cache replay --baseline main
```

## Everything you need

<div class="features">
	<div class="feature">
		<h3>Record / replay cassettes</h3>
		<p>Every model call is recorded to a local SQLite cassette keyed on a hash of the canonical request. Commit it and CI runs <code>--cache replay</code> byte-for-byte at $0, offline, with no API keys and no flaky judges.</p>
	</div>
	<div class="feature">
		<h3>Any language</h3>
		<p>Protocols over SDKs. Targets speak <code>openai-compatible</code>, <code>http</code>, or <code>shell</code>; custom scorers speak JSON over stdin/stdout; judges are any OpenAI-compatible endpoint. Rust is the engine — you never write Rust.</p>
	</div>
	<div class="feature">
		<h3>Agent trajectories</h3>
		<p>Feed recorded OTel / OpenInference traces to a <code>trace</code> target and grade the path with a <code>trajectory</code> scorer: <code>must_call</code> / <code>must_not_call</code> ordering, <code>max_steps</code> budgets, plus final-answer grading in the same suite.</p>
	</div>
	<div class="feature">
		<h3>Gates &amp; baselines</h3>
		<p>Block regressions, not imperfection. <code>--baseline</code> tolerates accepted failures and fails only on new ones; <code>run.gates</code> adds <code>pass_rate</code> and <code>mean_score</code> floors across the whole run.</p>
	</div>
	<div class="feature">
		<h3>Cost tracking &amp; budgets</h3>
		<p>Declare <code>cost</code> input/output rates and get per-case tokens and dollars. <code>run.budget_usd</code> stops scheduling new cases once the run hits its cap — replayed runs count their recorded virtual cost too.</p>
	</div>
	<div class="feature">
		<h3>HTML reports</h3>
		<p>Emit a single self-contained HTML file — every case, score, and diff inline, no assets to host. Drop it into CI artifacts and open the run in a browser.</p>
	</div>
</div>

## Works with your stack

<p class="stack">OpenAI, vLLM, Ollama, any OpenAI-compatible gateway, your own REST APIs, shell commands, and OTel / OpenInference traces.</p>
